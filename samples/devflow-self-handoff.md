## Title
Client handoff: DevFlow early access package

## TL;DR
DevFlow is a small command-line tool that turns recent git work into a **clear write-up you can send to a client or teammate** — without asking them to open your editor.

## Context
AI coding tools write code fast. Explaining that code to the next person is still slow. DevFlow produces a markdown file you can email, attach to an invoice, or paste into a pull request.

## What shipped / changed
- Installable CLI in our monorepo (`projects/devflow`)
- Commands: handoff, PR walkthrough, explain
- Modes for client, junior engineer, reviewer, or future-you
- Free to run with your own Anthropic API key
- Founding Pro listed at **$9/month** (limited seats)

## How it works (map)
1. You run one command in the project folder
2. DevFlow reads git history and the changed files (not your whole company drive)
3. It writes a structured markdown document on disk
4. You share that file — the value is the **file**, not a chat window

## Risks & landmines
- Without an API key you still get a useful outline (offline mode), but the narrative is thinner
- Very large changes may need a short note from you (`--extra`) so nothing important is missed
- This is beta software; treat outputs as drafts you review before sending to a paying client

## How to verify
- [ ] Open the generated `.md` file in any text editor
- [ ] Confirm the “what changed” list matches the work you intended
- [ ] Run the “how to verify” checklist on a staging environment
- [ ] Send the file to one real stakeholder and ask if they could act without a call

## Handoff checklist

- [ ] Sensitive paths/secrets removed
- [ ] Shared with next human
- [ ] Feedback captured (did they still need a Zoom?)
