## Title
PR #99 — Auth Icon Standardization & Layout Cleanup

## TL;DR
We replaced third-party social login icons with custom SVG icons and cleaned up responsive layout classes across login, signup, and password reset pages. The visual change is subtle but improves consistency and removes an external icon library dependency for auth screens. A background wrapper was added to key auth pages. No functional behavior changes.

## Context
This work continues a series of UI fixes on the `refactor/social-login-logo` branch. The team is standardizing how Kakao and Naver login icons render across breakpoints and reducing reliance on the `react-icons` package for brand assets that need exact sizing/color control.

## What shipped / changed
- **Custom SVG icons added**: `KakaoIcon` and `NaverIcon` components under `src/components/auth/` replace `react-icons/si` imports.
- **Icon sizing unified**: Login page social buttons now use `size-13.5` containers with 24px icons; signup start page uses `h-13.25` buttons with 18px Kakao / 16px Naver icons.
- **Layout wrapper added**: Login and signup start pages now wrap content in `<div className="min-h-dvh w-full bg-background">` to ensure consistent background color.
- **Responsive class cleanup**: Removed `sm:` breakpoint overrides from several auth pages (headings, padding, container widths). Pages now use a single mobile-first layout up to the `max-w-[440px]` container.
- **Metrics hook**: `metrics/events.jsonl` was added to record CLI handoff events (no user-facing impact).

## Risks & landmines
- **External dependency**: `react-icons/si` is no longer imported in these files, but the package may still be used elsewhere. Do not remove the dependency until all usages are migrated.
- **Icon color inheritance**: Icons use `fill="currentColor"`. If parent text colors change, icon colors will follow. Verify brand colors remain correct on colored buttons.
- **Layout shift on desktop**: Removing `sm:` padding/typography overrides means desktop and mobile now share the same spacing. If the design system expects larger type on desktop (`sm:text-[32px]`), this may look smaller than intended on wide screens.
- **Untracked file**: `metrics/events.jsonl` is untracked (`??`). Confirm with the team whether this should be committed, gitignored, or generated at build time.
